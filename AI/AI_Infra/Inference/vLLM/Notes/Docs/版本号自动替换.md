vllm_ascend_version 变量的工作原理
1. 变量定义位置
变量在 docs/source/conf.py 的第 64-81 行中定义：


myst_substitutions = {
    "vllm_ascend_version": "v0.14.0rc1",
    "vllm_version": "v0.14.1",
    "pip_vllm_ascend_version": "0.14.0rc1",
    "pip_vllm_version": "0.14.1",
    "cann_image_tag": "8.5.0-910b-ubuntu22.04-py3.11",
    ...
}
这是一个手动维护的字典，发版时需要手动更新。

2. 底层机制 — Sphinx + MyST 替换系统
整个流程依赖以下组件的协作：

组件	作用
myst_parser	让 Sphinx 能解析 Markdown（.md）文件
sphinx_substitution_extensions	扩展 Sphinx 的代码块以支持变量替换
myst_enable_extensions = ["substitution"]	在 MyST 中启用 `
3. 使用方式
在 Markdown 文件中，代码块需要声明 :substitutions: 指令才能触发替换：


   :substitutions:

export IMAGE=quay.io/ascend/vllm-ascend:|vllm_ascend_version|
构建文档时，|vllm_ascend_version| 会被替换为 v0.14.0rc1，最终渲染为：


export IMAGE=quay.io/ascend/vllm-ascend:v0.14.0rc1
4. 数据流总结

conf.py
  └─ myst_substitutions["vllm_ascend_version"] = "v0.14.0rc1"
          │
          ▼ Sphinx 构建时注入
quick_start.md (有 :substitutions: 的代码块)
          │
          ▼ |vllm_ascend_version| 被替换
最终 HTML：quay.io/ascend/vllm-ascend:v0.14.0rc1
注意：这个变量是静态硬编码的，不会自动从 pyproject.toml 或 version.py 中读取，每次发版需要人工同步更新 conf.py。
